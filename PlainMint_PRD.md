## 1. 文档说明

### 1.1 编制目的

本文档用于明确 PlainMint 轻量化跨平台纯文本编辑器的产品定位、目标用户、功能边界、交互逻辑、界面规范、数据结构、异常处理、非功能要求及版本规划，为产品设计、UI 设计、研发实现、测试验收和后续迭代提供统一依据。

### 1.2 产品定义

PlainMint 是一款面向 Windows 和 macOS 的轻量化纯文本编辑器，定位介于系统自带记事本和 Notepad++ 之间。

名称由 **Plain Text** 与 **Mint** 组合而来，表达“纯文本、清爽编辑”的产品理念。产品英文宣传语建议使用：**Plain text, freshly simple.**

产品不追求复杂的代码编辑、插件扩展和开发工具能力，而是围绕以下核心价值进行设计：

1. 打开快、编辑流畅。
2. 多文件管理简单。
3. 自动备份可靠。
4. 文本搜索、替换和分屏查看方便。
5. 支持将文件保存至 OneDrive、iCloud Drive、Dropbox 等本地同步目录。
6. 界面简洁、现代、优雅，不臃肿。
7. Windows 与 macOS 功能一致，并适配各自操作习惯。

### 1.3 产品原则

- **纯文本优先**：只处理纯文本，不引入富文本格式。
- **可靠优先**：自动备份、异常恢复和保存安全高于功能数量。
- **轻量优先**：不加入文件树、终端、插件中心、Git、调试器等 IDE 功能。
- **本地优先**：文件以本地文件系统为核心，云同步由本地网盘目录承接。
- **少即是多**：高频功能直接可见，低频功能收纳到菜单或设置中。
- **跨平台一致**：功能结构一致，窗口控制、快捷键和系统对话框遵循平台规范。
- **用户可控**：自动保存、备份周期、主题色、字体、行号、换行等均可配置。

## 2. 产品背景

现有桌面文本编辑工具主要存在以下问题：

1. 系统记事本功能过少，多文件、自动备份、分屏等能力不足。
2. Notepad++、VS Code 等工具功能过多，普通用户使用时存在明显学习成本。
3. 部分跨平台编辑器体积较大、启动较慢，界面偏开发工具风格。
4. 云同步产品往往依赖账号体系和专有格式，不适合普通纯文本文件长期保存。
5. 用户在编辑日志、配置文件、文案、会议记录、清单时，需要一个更可靠但仍然简单的桌面工具。

PlainMint 的机会在于：以轻量化体验为基础，补齐系统记事本缺失的高频功能，同时避免演变成代码 IDE。

## 3. 产品目标

### 3.1 核心目标

1. 提供稳定、快速、无干扰的纯文本编辑体验。
2. 提供多 Tab、左右分屏、搜索替换、自动备份和会话恢复等核心增强能力。
3. 支持本地文件和网盘同步目录中的文件。
4. 提供简洁、年轻化、现代化的桌面 UI。
5. 提供 5 种内置主色调，用户可自由切换。
6. 支持应用版本自动检查和更新。
7. 在 Windows 和 macOS 上保持主要功能一致。

### 3.2 体验目标

- 冷启动后快速进入可编辑状态。
- 新用户无需阅读帮助即可完成新建、打开、编辑、保存。
- 用户未主动保存时，异常退出后仍可恢复大部分编辑内容。
- 用户可在 2 次以内点击完成查找、换行和分屏等高频操作。
- 主界面不出现左侧文件树、复杂面板或大量低频按钮。

### 3.3 非目标

首版不建设以下能力：

- 代码自动补全。
- 语法检查与代码诊断。
- 插件系统。
- 终端和命令行。
- Git 集成。
- FTP/SFTP。
- 数据库连接。
- 宏录制。
- 富文本编辑。
- 在线多人协作。
- 独立云账号体系。
- AI 写作或 AI 编程。
- 无限分屏。
- 项目文件夹管理。
- 复杂正则表达式构建器。

## 5. 典型使用场景

### 5.1 临时记录

用户启动 PlainMint，新建空白文件并输入内容，应用自动生成恢复副本。若文件为空，关闭时直接退出且不显示确认；若文件包含未保存内容，用户可选择保存、放弃或取消关闭。

### 5.2 多文件并行编辑

用户同时打开多个 TXT、LOG、MD 文件，通过 Tab 快速切换。未保存文件以圆点标识。

### 5.3 左右对照查看

用户将当前文件或另一个文件拖动到右侧，进入左右分屏模式。两侧可独立滚动、定位和切换 Tab。

### 5.4 网盘同步

用户在设置中选择 OneDrive 本地同步目录作为云同步目录。PlainMint 将新文件优先保存到该目录，网盘客户端负责同步。

### 5.5 异常恢复

应用意外关闭后再次启动，检测到未完成会话，提示恢复未保存内容和上次打开的文件。

### 5.6 日志查看

用户打开较长日志文件，关闭自动换行，使用行号和搜索功能定位问题。

### 5.7 文本批量替换

用户打开查找替换栏，输入关键词和替换内容，逐个替换或全部替换，并可通过撤销恢复。

## 7. 主界面设计

### 7.1 总体布局

```text
┌──────────────────────────────────────────────────────────────┐
│ 应用图标  PlainMint                         最小化 最大化 关闭 │
├──────────────────────────────────────────────────────────────┤
│ 新建  打开  保存  撤销  重做  查找  换行  分屏  更多       │
├──────────────────────────────────────────────────────────────┤
│ readme.txt ×   todo.txt ● ×   notes.md ×   ＋               │
├──────────────────────────────────────────────────────────────┤
│  1  文本内容                                                  │
│  2                                                           │
│  3                                                           │
│                                                              │
├──────────────────────────────────────────────────────────────┤
│ 第3行，第1列  选中0字符  共356字符  UTF-8  LF  纯文本       │
└──────────────────────────────────────────────────────────────┘
```

### 7.2 布局原则

- 不设置左侧导航栏。
- 不默认显示文件树。
- 不使用大面积卡片布局。
- 工具栏仅放置高频操作。
- 状态栏使用弱视觉层级。
- 编辑区为视觉焦点。
- 主界面使用统一的跨平台品牌化无边框窗口，不模仿任一操作系统皮肤。
- 工具栏、Tab 栏和状态栏高度保持紧凑。

主界面采用“PlainMint Minimal”紧凑密度：标题栏约 46px、工具栏约 48px、Tab 栏约 40px、状态栏约 30px。后续调整应优先保障编辑区高度，不因装饰性留白压缩核心内容区域。

### 7.3 工具栏

| 功能 | 说明 |

| 保存 | 保存当前文件 |
| 撤销 | 撤销当前文档操作 |
| 重做 | 重做当前文档操作 |

工具栏要求：

- 新建、打开、保存等核心操作图标和文字同时显示；辅助操作使用图标并提供功能与快捷键提示。
- 在 720px 最小窗口宽度下保持工具栏单行，不依赖新增的溢出菜单。
- 当前开启的“换行”“分屏”等功能应显示选中状态。
- 不使用超过 2 种强调色。
- 工具栏按钮支持鼠标悬停提示和快捷键提示。

## 8. 功能需求

# 8.1 纯文本编辑

### 8.1.1 功能说明

支持新建、打开、编辑、保存、另存为纯文本文件。

### 8.1.2 支持文件类型

首版按纯文本方式打开以下扩展名：

- `.txt`
- `.log`
- `.md`
- `.json`
- `.xml`
- `.csv`
- `.ini`
- `.conf`
- `.yaml`
- `.yml`
- `.properties`
- `.sql`
- 无扩展名文件

不对文件内容进行富文本渲染。

### 8.1.3 编辑能力

- 文本输入。
- 光标移动。
- 文本选择。
- 剪切、复制、粘贴。
- 全选。
- 删除。
- 撤销、重做。
- 支持系统输入法。
- 支持拖放文本。
- 支持标准文本快捷键。
- 支持水平和垂直滚动。
- 支持 Tab 字符输入。
- 支持设置 Tab 宽度。
- 支持普通选择、矩形选择和多光标选择。
- 支持当前词与选中词的全部匹配高亮。
- 支持跳转到指定行。
- 支持行移动、行复制、行删除，以及 Tab / Shift+Tab 缩进与反缩进。
- 支持显示不可见字符和特殊空白字符。
- 支持系统拼写检查开关，默认关闭。
- 自动缩进开关不纳入 V1.0，避免在纯文本、日志与配置文件中引入非预期空白字符。

### 8.1.4 CodeMirror 能力边界与后续规划

PlainMint 使用 CodeMirror 6 作为编辑器内核，但不以“把编辑器能力全部暴露”为目标。功能只有在不破坏纯文本、轻量和可靠性原则时才进入产品路线图。

| 类别 | 能力 | 需求与依赖 | 产品决策 |
|---|---|---|---|
| 选择与光标 | 多光标、矩形选择、选择下一个匹配、当前词匹配高亮 | 使用现有 CodeMirror 选择与搜索能力 | 保持键盘优先，不增加常驻工具栏按钮 |
| 导航与行编辑 | 跳转行、移动/复制/删除当前行、缩进/反缩进、撤销/重做 | 使用现有 CodeMirror 命令集 | 在设置的“快捷键说明”页维护快捷键速查 |
| 搜索 | 正则搜索、捕获组替换、选中内容带入查找、全部匹配一次多选 | 使用 `@codemirror/search` | 不新增依赖 |
| 文本可见性 | 特殊空白字符显示、拼写检查、当前行高亮、行号、自动换行 | 使用编辑器视图扩展与系统拼写服务 | 默认行为需以普通写作可读性为准 |
| 文件对比 | 双栏 Merge View、差异块导航、冲突内容对照 | 使用 `@codemirror/merge` | 仅在分屏模式下提供左右活动文档的只读对比；不提供合并写入 |
| 语法高亮 | 自动识别与手动选择语言模式；支持纯文本、Markdown、JSON/JSONC、YAML、XML、HTML、CSS/SCSS/Less、JavaScript/JSX、TypeScript/TSX、Python、SQL、Shell、PowerShell、C/C++、C#、Java、Go、Rust、PHP | 使用 `@codemirror/language`、官方语言包与官方 legacy modes；文件名优先、未知扩展名仅用高置信内容特征辅助识别 | 手动模式随工作区会话保存，不改变 dirty、撤销或文本内容；新建无扩展名文件空闲后只识别一次；超过 5 MiB 或 50,000 行暂停解析；不纳入补全、诊断、格式化或折叠界面 |
| 补全与诊断 | 自动补全、拼写/语法诊断、Lint 标记、快速修复 | 需要引入 `@codemirror/autocomplete`、`@codemirror/lint` 及语言服务 | 不纳入产品范围；不得在无明确规则来源时给用户伪诊断 |

规则：新增编辑器扩展必须支持双语文案、键盘操作、只读文档保护、分屏一致性、撤销语义和大文本回归测试。涉及语言包、网络词典、分析服务或可能改变文本内容的扩展，必须单独进行产品评审。

### 8.1.5 文件保存

保存流程：

1. 已有路径：写入原文件。
2. 新建文件：弹出系统保存对话框；未命名文档以“未命名 1、未命名 2……”区分，首次保存默认文件名为 `untitled-1.txt`、`untitled-2.txt`。
3. 保存成功：清除未保存状态。
4. 保存失败：保留编辑内容并提示错误。
5. 文件被外部修改：进入冲突处理流程。

### 8.1.6 安全写入

为降低断电或异常退出造成的文件损坏风险，建议采用原子写入：

1. 将内容写入同目录临时文件。
2. 确认写入完成。
3. 替换原文件。
4. 写入失败时保留原文件。

# 8.2 多 Tab

### 8.2.1 功能说明

用户可同时打开多个文件，每个文件对应一个 Tab。

### 8.2.2 Tab 状态

| 状态 | 展示方式 |
|---|---|
| 当前 Tab | 主色底部指示条或浅色背景 |
| 未保存 | 文件名右侧显示主色圆点 |
| 只读 | 显示锁定图标 |
| 文件丢失 | 显示警示图标 |
| 外部修改 | 显示更新提示图标 |

### 8.2.3 Tab 操作

- 单击切换。
- 中键关闭。
- 点击关闭按钮关闭。
- 拖动排序。
- 拖到右侧进入分屏。
- 右键关闭当前。
- 右键关闭其他。
- 右键关闭右侧。
- 右键复制文件路径。
- 重新打开最近关闭的 Tab。
- 双击 Tab 空白区新建文件。

最近关闭记录仅保留有路径文件，按路径去重并持久化最近 10 条；重新打开时读取磁盘中的最新内容。若文件已在任一栏打开，则直接激活已有 Tab。

### 8.2.4 Tab 关闭逻辑

关闭逻辑按内容处理：

- 文档内容长度为 0 时直接关闭，不弹出确认。
- 仅包含空格、制表符或换行的文档仍视为有内容。
- 存在未保存内容且文档非空时，弹出确认：
  - 保存并关闭。
  - 不保存并关闭。
  - 取消。

多个未保存文件同时退出时，使用统一列表弹窗批量处理。

“关闭其他”和“关闭右侧”涉及多个未保存文档时同样使用统一列表弹窗；只有全部保存成功后才执行关闭，任何保存失败或用户取消都会保留原有 Tab。

# 8.3 自动备份

### 8.3.1 功能说明

自动备份用于保存恢复副本，不直接覆盖原文件。

### 8.3.2 默认规则

- 默认开启。
- 停止输入 3 秒后触发。
- 同一文档连续输入时采用防抖机制。
- 新建未保存文件同样生成备份。
- 默认保留 30 天。
- 每个文件默认最多保留 20 个历史版本。
- 超出保留周期或数量后自动清理。

### 8.3.3 备份内容

- 文档内容。
- 原文件路径。
- 文件名。
- 备份时间。
- 文件编码。
- 换行格式。
- 光标位置。
- 编辑器滚动位置。
- 文档是否为未保存文件。

### 8.3.4 备份目录

备份文件存储于应用数据目录，不默认存放在原文件目录。

### 8.3.5 恢复入口

- 应用异常启动恢复提示。
- 设置中的“备份与恢复”页面。
- 文件菜单中的“恢复历史版本”。

# 8.4 自动保存

### 8.4.1 功能说明

自动保存将编辑内容直接写入原文件，与自动备份不同。

### 8.4.2 默认状态

默认关闭。

### 8.4.3 可选策略

- 从不自动保存。
- 停止输入 10 秒后保存。
- 每 30 秒保存。
- 应用失去焦点时保存。
- 切换 Tab 时保存。

### 8.4.4 限制

- 新建但未指定路径的文件不执行自动保存。
- 外部修改冲突时暂停自动保存。
- 只读文件不执行自动保存。
- 保存失败后停止连续重试，并提示用户。
- 同一内容版本失败后不再自动重试；用户继续编辑后恢复一次自动尝试，也可手动重试或另存为。

# 8.6 网盘同步目录

### 8.6.1 产品策略

首版不直接集成 OneDrive、Dropbox、Google Drive、iCloud Drive 的账号授权 API。

采用本地同步目录模式：

- 用户选择已被网盘客户端同步的本地目录。
- PlainMint 将其作为普通本地目录读写。
- 网盘客户端负责上传和下载。

### 8.6.2 支持方式

设置中提供：

- 默认保存目录。
- 云同步目录。
- 最近使用目录。
- 恢复为系统文档目录。

### 8.6.3 云同步目录识别

应用仅校验：

- 目录是否存在。
- 是否可读。
- 是否可写。
- 可用空间是否足够。

应用不保证云服务当前在线。

无路径新文件首次保存时优先使用云同步目录；云同步目录不可用时依次回退到默认保存目录和系统保存位置。目录在使用前会重新校验。现有文件“另存为”仍从原文件位置开始。

### 8.6.4 冲突处理

文件被网盘或其他程序修改时：

**当前文档无未保存修改：**

- 检测到指纹变化后重新读取磁盘内容，并在文档仍处于同一干净修订时自动重新加载。
- 自动清空撤销/重做历史、更新指纹并提示“文件已更新”。如果读取期间产生了本地编辑，则绝不覆盖编辑，转为冲突状态。

**当前文档有未保存修改：**

显示常驻冲突提示；只有用户点击“处理冲突”或手动保存命中指纹冲突时才打开专用弹窗。弹窗使用并排只读差异视图；单侧超过 5 MiB 或 50,000 行时降级为并排纯文本视图。

- **稍后处理**：关闭弹窗并保留冲突状态，自动保存继续暂停。
- **将当前版本另存为副本**：必须选择与原文件不同的路径；成功后当前文档转为该新文件。
- **重新加载磁盘版本**：再次读取并确认磁盘版本未变化；先强制备份当前本地编辑，再替换编辑器内容。
- **覆盖磁盘版本**：必须经过二次确认；再次读取并确认磁盘版本未变化，先强制备份磁盘版本，再以该指纹为前置条件保存本地编辑。

冲突安全备份不受“自动备份”开关影响，但仍使用已有的保留天数和单文件版本数限制；恢复中心会展示“本地编辑冲突”或“磁盘版本冲突”的备份来源。

首版不提供自动三方合并。

# 8.15 外部文件变化检测

Windows 使用文件系统事件驱动检测：应用非递归监听每个已打开文件的父目录，以覆盖 OneDrive 等同步工具通过临时文件替换、移动或重命名更新目标文件的情况。相关事件按目标文件独立防抖 750ms，防抖结束后才读取文件并计算指纹。

所有平台保留每 60 秒一次的低频兜底，但兜底只读取修改时间、文件大小、存在性和只读状态；只有修改时间或大小发生变化时才继续读取内容并计算哈希。监听不可用或单个目录注册失败时静默使用该兜底策略。macOS 应采用同等的目录事件监听策略，以覆盖原子替换、移动和重命名。

### 8.15.1 检测内容

- 修改时间变化。
- 文件大小变化。
- 文件被删除。
- 文件被重命名。
- 文件权限变化。

### 8.15.2 处理规则

| 状态 | 处理 |
|---|---|
| 无本地修改，磁盘已更新 | 提示重新加载，可配置自动加载 |
| 有本地修改，磁盘已更新 | 显示冲突提示 |
| 文件被删除 | 保留内容，标记文件不存在 |
| 文件变为只读 | 禁止覆盖保存，可另存为 |
| 文件恢复出现 | 提示重新关联 |

# 8.19 文本文件对比

### 8.19.1 入口与范围

- 仅在分屏模式且左右面板都有活动文档时，工具栏“对比”与 `Ctrl/Cmd+Shift+D` 可用。
- 左侧和右侧分别使用对应面板活动文档的当前内容快照，包含尚未保存的修改。
- 对比不会读取额外磁盘文件，不新增 Tab、不写入最近文件、不加入会话恢复，也不建立外部变更监听。
- 左右两侧为同一文档时允许打开对比，并显示无差异结果。

### 8.19.2 展示与导航

- 两侧均只读，使用双栏差异视图高亮文本变更。
- 显示左右两侧文件名、路径、编码和换行符信息。
- 提供上一处、下一处差异按钮；差异导航在首尾循环。
- 无差异时明确显示“没有文本差异”。
- 任一侧超过 5 MiB 或 50,000 行时，降级为并排只读文本，不计算高亮差异。

### 8.19.3 边界

- 对比窗口不提供接受、拒绝、编辑、合并或写回文件操作。
- 对比结果不会随编辑器内容自动刷新；用户重新发起对比以获取新快照。

## 17. 性能要求

### 17.1 启动性能

目标值：

- 冷启动进入可编辑状态：尽量控制在 1 秒内。
- 热启动：尽量控制在 500ms 内。
- 不等待更新检查完成后再显示界面。

### 17.2 编辑性能

- 10MB 以内文本输入无明显延迟。
- 20MB 文本滚动保持基本流畅。
- 查找过程不阻塞主线程。
- 字符统计采用增量计算或合理节流。
- 自动备份不得明显影响输入。

### 17.3 内存

- 空白启动保持较低内存占用。
- 关闭 Tab 后释放文档和编辑器资源。
- 超大文档限制撤销历史。
- 分屏同一文档时不得复制完整内容。

## 18. 安全要求

- 文件访问范围遵循用户主动选择。
- 不后台扫描用户目录。
- 更新包必须经过签名验证。
- 临时备份文件应使用不可预测标识。
- 备份文件不主动暴露到共享目录。
- 保存失败不得破坏原文件。
- 删除备份需要二次确认。
- 错误日志不得记录用户正文。
- 外部 URL 仅通过系统默认浏览器打开。

## 19. 无障碍要求

- 支持键盘完成主要操作。
- 所有图标按钮提供文本标签或可访问名称。
- Tab、按钮、开关具备明确焦点状态。
- 主色调不作为唯一状态表达方式。
- 文字与背景对比度满足基本可读性。
- 支持系统缩放。
- 支持高 DPI。
- 支持屏幕阅读器读取菜单和设置项。
- 动画尽量简短，并尊重系统减少动画设置。

## 23. 验收标准

### 23.1 基础编辑

- 能正确新建、打开、编辑和保存纯文本文件。
- 保存后文件内容与编辑区一致。
- 保存失败时编辑内容不丢失。
- 支持 UTF-8、UTF-8 BOM、UTF-16 LE、UTF-16 BE。
- 能正确保留原换行格式。

### 23.2 多 Tab

- 可同时打开不少于 20 个 Tab。
- Tab 可切换、排序和关闭。
- 未保存状态展示准确。
- 切换 Tab 后内容和光标位置正确。
- 内容长度为 0 的未保存文档关闭时不弹出确认。
- 含空格、制表符、换行或正文的未保存文档关闭时必须提示保存、放弃或取消。

### 23.3 自动备份

- 停止输入后能按设置生成备份。
- 异常退出后可恢复未保存内容。
- 正常保存不影响备份历史。
- 超出保留规则后能自动清理。

### 23.4 分屏

- 可在单栏和双栏之间切换。
- 左右栏可显示不同文件。
- 同一文件可在两侧同时显示。
- 两侧滚动和光标互不干扰。
- 同一文件内容保持同步。

### 23.5 搜索替换

- 能定位所有匹配项。
- 匹配数量显示正确。
- 区分大小写和全字匹配有效。
- 全部替换可一次撤销。
- 无匹配时提示清晰。

### 23.6 主题色

- 设置中显示 5 种内置色调。
- 切换后立即生效。
- 重启后保持用户选择。
- 浅色和深色模式均可正常显示。
- 主色不影响错误、警告、成功等语义色。
- 当前 Tab、按钮、开关和活动分屏使用统一主色。

### 23.7 网盘目录

- 可选择本地同步目录。
- 可正常保存到 OneDrive 等本地目录。
- 目录不可写时有清晰提示。
- 外部同步修改可被检测。

### 23.8 更新

- 能检测新版本。
- 能展示更新说明。
- 更新失败不影响当前版本使用。
- 下载包签名校验失败时禁止安装。

### 23.9 跨平台

- Windows 和 macOS 核心功能一致。
- 快捷键符合各自平台习惯。
- 系统打开、保存对话框正常。
- 高 DPI 和不同分辨率下界面不变形。

- 稳定版。
- 可评估测试版。
- Windows GitHub Release 必须生成 MSI/NSIS 的 `.sig` 与 `latest.json`，稳定版 Windows 客户端通过固定公钥验证后安装。
- macOS 必须发布自动更新压缩包、签名与 `darwin-*` 更新清单，并完成升级验证。
- 更新签名私钥仅保存在 GitHub Actions Secrets 和维护者的加密离线备份中，不进入仓库。

### 25.4 版本号

采用语义化版本：

```text
主版本.次版本.修订版本
例如：1.2.3
```

- 手动发布工作流以仓库中最新版本为基准自动递增。
- 应用源码中的版本号由发布工作流同步，无需人工逐文件修改。
- 正式版本 Tag 统一使用 `v主版本.次版本.修订版本`，例如 `v1.2.3`。

## 27. 最终产品边界

PlainMint V1.0 应聚焦于以下核心闭环：

```text
打开文件
→ 多 Tab 编辑
→ 查找替换与分屏查看
→ 自动备份
→ 本地或网盘目录保存
→ 异常恢复
→ 持续更新
```

产品的差异化不在于功能数量，而在于：

- 比系统记事本更完整。
- 比 Notepad++ 更简洁。
- 比 IDE 更轻量。
- 比在线笔记更开放。
- 比普通文本工具更可靠。
- 通过 5 种内置主色调提供个性化但克制的视觉体验。
